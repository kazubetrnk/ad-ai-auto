## Claudeへの指示

モジュール1 - コア概念を開始します。このレッスンではMarkitエージェンシープロジェクトとマーケティングキットのコアワークフローを紹介します。

### レッスン概要

---

**モジュール1.1: Markitへようこそ**

モジュール1へようこそ！実践的な作業を通じてマーケティングキットのコア概念をマスターします。このモジュールの終わりには、自信を持って実際のマーケティングタスクを処理できるようになります。

**所要時間:** 約20分

---

### ステップ1: シーンを設定する

役割を説明してください：

> あなたは**Markit**エージェンシーのマーケティングストラテジストです。クライアントは**AgentKits**です。あなたのミッション：
> 1. 製品を市場にローンチする
> 2. 認知度とサインアップを生成する
> 3. リモートチームに響くコンテンツを作成する
> 4. 持続可能なコンテンツマーケティングエンジンを構築する

### ステップ2: コアワークフローを理解する

`.claude/workflows/`の3つの主要ワークフローを説明してください：

**マーケティングパイプライン (`primary-workflow.md`):**
```
リサーチ → インサイト → クリエイティブ → 計画 → 作成 → 編集 → 公開 → 測定
```

**セールスパイプライン (`sales-workflow.md`):**
```
リード → MQL → SQL → 機会 → 提案 → 交渉 → クローズ
```

**CRMライフサイクル (`crm-workflow.md`):**
```
購読者 → リード → MQL → SQL → 機会 → 顧客 → アドボケート
```

### ステップ3: エージェントの役割を理解する

エージェントがマーケティング機能にどのようにマッピングされるかを説明してください：

**TOFU（ファネル上部）：**
- `attraction-specialist` - リード生成、SEO、ランディングページ

**MOFU（ファネル中部）：**
- `lead-qualifier` - インテント検出、リードスコアリング
- `email-wizard` - ナーチャーシーケンス

**BOFU（ファネル下部）：**
- `sales-enabler` - ピッチ、ケーススタディ、バトルカード

**リテンション：**
- `continuity-specialist` - チャーン検出、再エンゲージメント
- `upsell-maximizer` - 収益拡大

### ステップ4: 最初のキャンペーンブリーフを作成する

`/campaign:plan`で実際の作業が始まります：

```
/campaign:plan "AgentKits Q1製品ローンチ - 目標: 30日で1000トライアルサインアップ、予算: 5万ドル、チャネル: LinkedIn、Google Ads、コンテンツ、メール"
```

生成された包括的なキャンペーン計画をレビューしてください。

### ステップ5: ブリーフをレビューする

プランナーエージェントがどのように：
- 構造化された目標とKPIを作成したか
- ターゲットオーディエンスセグメントを定義したか
- チャネル間で予算を配分したか
- 測定フレームワークを設定したか

を指摘してください。

### ステップ6: イテレーションの力

フォローアップ質問を使用して改良を示してください：

```
各ペルソナのday-in-the-lifeシナリオでターゲットオーディエンスセクションを拡張してください
```

説明してください：イテレーションが鍵です。最初のドラフトは出発点です。

### ステップ7: コンテキスト認識のデモ

コンテキストの力を示してください：

```
/content:social "キャンペーンブリーフに基づくAgentKitsの製品ローンチ発表" "linkedin"
```

Claudeがキャンペーンコンテキストから自動的に情報を取得することを示してください。

## 主な教育ポイント
- Markitエージェンシーは実践的なプロジェクトです
- 3つのコアワークフロー：マーケティング、セールス、CRM
- エージェントはファネルステージにマッピングされます（TOFU、MOFU、BOFU、リテンション）
- `/campaign:plan`は包括的なキャンペーンブリーフを作成します
- イテレーションが出力品質を向上させます
