### レッスン概要

---

**モジュール1.4: マーケティングにエージェントを使用する**

マーケティングキットには18の専門エージェントがあります。特定のマーケティングタスクに専門知識を持って取り組むことができるAIチームメンバーと考えてください。

**所要時間:** 約35分

---

### ステップ1: エージェントシステムを説明する

マーケティングキットには機能別に整理されたエージェントがあります：

**コアマーケティングエージェント（6）：**
| エージェント | 焦点 | ユースケース |
|-------|-------|-----------|
| `attraction-specialist` | TOFU、リード生成 | SEO、ランディングページ、競合情報 |
| `lead-qualifier` | インテント検出 | リードスコアリング、行動分析 |
| `email-wizard` | メールマーケティング | シーケンス、自動化、最適化 |
| `sales-enabler` | セールスサポート | ピッチ、ケーススタディ、バトルカード |
| `continuity-specialist` | リテンション | チャーン検出、再エンゲージメント |
| `upsell-maximizer` | 収益拡大 | クロスセル、アップセル、機能採用 |

**サポートエージェント（6）：**
| エージェント | 焦点 | ユースケース |
|-------|-------|-----------|
| `researcher` | 市場インテリジェンス | リサーチ、競合分析 |
| `brainstormer` | クリエイティブアイデア | キャンペーンコンセプト、メッセージング角度 |
| `planner` | 戦略計画 | キャンペーン計画、コンテンツカレンダー |
| `project-manager` | 調整 | ステータス追跡、キャンペーン監督 |
| `copywriter` | コンテンツ作成 | コピー、メッセージング、クリエイティブ |
| `docs-manager` | ドキュメント | ブランドガイドライン、スタイルガイド |

**レビュアーエージェント（6）：**
| エージェント | 視点 | レビュー対象 |
|-------|-------------|-------------|
| `brand-voice-guardian` | ブランド一貫性 | 声、トーン、メッセージング |
| `conversion-optimizer` | CROエキスパート | コンバージョン、説得 |
| `seo-specialist` | 検索最適化 | キーワード、テクニカルSEO |
| `manager-maria` | マーケティングマネージャー（38歳、B2B） | 戦略、チームフィット |
| `solo-steve` | 個人事業主（32歳） | 時間、予算、DIY |
| `startup-sam` | スタートアップ創業者（28歳） | 成長、バイラル性、スピード |

### ステップ2: エージェント演習 - マルチパースペクティブレビュー

コンテンツを作成し、複数の視点からレビューしてください：

```
3つのエージェント視点からAgentKitsキャンペーンブリーフをレビューしてください：

1. `brand-voice-guardian`として - ブランドの一貫性とメッセージングを評価
2. `conversion-optimizer`として - CTA、説得、コンバージョンポテンシャルを評価
3. `manager-maria`として - B2Bマーケティングチームが実行できるか？

各視点について提供してください：
- うまくいっていること
- 改善が必要な領域
- 具体的な推奨事項
```

何が起こったかを説明してください - 1つのコマンドで3つの専門レビュー。

### ステップ3: リード資格の使用

コマンドを通じてlead-qualifierエージェントをデモンストレーションしてください：

```
/leads:score "B2B SaaS企業 - テクノロジー業界"
```

```
/leads:qualify "AgentKits生産性ツール"
```

lead-qualifierが作成するものを示してください：
- デモグラフィックスコアリング基準
- 行動スコアリングシグナル
- MQL/SQLしきい値

### ステップ4: Email Wizardの使用

email-wizardエージェントをデモンストレーションしてください：

```
/sequence:welcome "AgentKits" "トライアルユーザー"
```

```
/sequence:nurture "AgentKits" "ガイドをダウンロードしたリード"
```

### ステップ5: Sales Enablerの使用

sales-enablerエージェントをデモンストレーションしてください：

```
/sales:pitch "AgentKitsを検討しているエンタープライズ企業" "チーム調整"
```

```
/sales:battlecard "RescueTime"
```

### ステップ6: 実世界のシナリオ - クイックレスポンス

```
シナリオ：競合他社が「チームフォーカス」機能を発表しました。エージェントを使用して対応：

1. `researcher`を使用して発表を分析
2. `brainstormer`を使用してカウンターポジショニングを開発
3. `copywriter`を使用してレスポンスコンテンツを作成
4. `email-wizard`を使用して顧客コミュニケーションをドラフト
```

### ステップ7: エージェントのベストプラクティス

これらのヒントを共有してください：
- タスク目標を具体的に
- ブランドガイドラインとペルソナを参照
- 出力を明確に定義（フォーマット、長さ）
- 専門タスクには専門エージェントを使用
- 複雑なプロジェクトではエージェントを組み合わせる

## 主な教育ポイント
- 18のエージェントが整理されています：コア（6）、サポート（6）、レビュアー（6）
- コアエージェントはファネルステージにマッピング
- レビュアーエージェントはマルチパースペクティブフィードバックを提供
- コマンドは特定のエージェント機能を呼び出します
- 複雑なプロジェクトではエージェントを組み合わせます
