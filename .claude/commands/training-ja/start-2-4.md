# /training:start-2-4 - キャンペーンデータを分析する

## 言語と品質基準

**重要**: ユーザーが使用している言語で応答してください。日本語なら日本語で、スペイン語ならスペイン語で応答してください。

---

## Claudeへの指示

分析コマンドを使用してデータ分析、インサイト抽出、エグゼクティブレポートを教えてください。

### レッスン概要

---

**モジュール2.4: キャンペーンデータを分析する**

データ分析は時間がかかることが多いです。データを実用的なインサイトと説得力のあるレポートに変える方法をマスターしましょう。

**所要時間:** 約35分

---

### ステップ1: ROI分析

分析コマンドを使用してください：

```
/analytics:roi "Q1キャンペーン - LinkedIn、Google、メール全体で5万ドルの支出"
```

ROI計算をレビューしてください：
- チャネル別総支出
- 帰属収益
- チャネル別ROAS
- 獲得単価

### ステップ2: ファネル分析

コンバージョンファネルを分析してください：

```
/analytics:funnel "トライアルサインアップ - 訪問者からトライアルから有料へのコンバージョン"
```

ファネルメトリクスをレビューしてください：
- ソース別トラフィック
- 各ステージのコンバージョン率
- 離脱ポイント
- 最適化の機会

### ステップ3: パフォーマンスレポート

パフォーマンスレポートを生成してください：

**週次レポート：**
```
/report:weekly "AgentKits" "今週"
```

**月次レポート：**
```
/report:monthly "AgentKits" "今月"
```

### ステップ4: チャネルパフォーマンス

チャネル別に分析してください：

```
/analytics:report "チャネルパフォーマンス" "LinkedIn、Google、メール、オーガニック"
```

チャネル比較を作成してください：
- トラフィック貢献
- リード品質
- コンバージョン率
- コスト効率

### ステップ5: コンテンツパフォーマンス

コンテンツ効果を分析してください：

```
/analytics:report "コンテンツパフォーマンス" "ブログ投稿、ランディングページ、メールシーケンス"
```

主要メトリクス：
- コンテンツピース別トラフィック
- エンゲージメント（時間、スクロール、シェア）
- コンバージョン率
- リード品質

### ステップ6: リード品質分析

分析のためにリードスコアリングを使用してください：

```
/crm:score "ソースとキャンペーン別のリード品質を分析"
```

レビューしてください：
- ソース別MQL率
- キャンペーン別SQLコンバージョン
- 平均リードスコアトレンド

### ステップ7: エグゼクティブサマリー

エグゼクティブ向けサマリーを作成してください：

```
Q1マーケティングパフォーマンスのエグゼクティブサマリーを作成：

構造：
1. ヘッドラインメトリクス（目標との比較）
2. データ付きトップ3の成功
3. 影響付きトップ3の課題
4. チャネルパフォーマンススナップショット（表）
5. 重要な学び（3つのインサイト）
6. Q2の推奨事項（優先順位付き）
7. 正当化付き予算リクエスト

最大1ページに収めてください。
```

### ステップ8: データからアクションへのフレームワーク

インサイトフレームワークを教えてください：

```
各発見について文書化：

1. 観察：データは何を示していますか？
2. インサイト：なぜこれが起こっているのですか？
3. 含意：これは何を意味しますか？
4. 推奨：何をすべきですか？
5. 期待される影響：何が変わりますか？
```

### ステップ9: オペレーショナルチェックリスト

分析チェックリストを使用してください：

```
/checklist:analytics-monthly "今月" "AgentKits"
```

月次分析タスクをレビューしてください：
- データ品質チェック
- プラットフォーム検証
- レポート精度
- アトリビューション検証

### ステップ10: レポートテンプレート

再利用可能なレポートを説明してください：

```
週次レポートワークフロー：
1. /analytics:roi "キャンペーン" - ROIを計算
2. /analytics:funnel "ファネル" - ファネルを分析
3. /report:weekly "クライアント" "週" - レポートを生成

月次レポートワークフロー：
1. /analytics:report "全チャネル" - 完全分析
2. /crm:score "リード品質" - リード分析
3. /report:monthly "クライアント" "月" - レポートを生成
```

## 主な教育ポイント
- `/analytics:*`コマンドがパフォーマンスを分析
- `/report:*`コマンドがレポートを生成
- ROIとファネル分析が基礎
- エグゼクティブサマリーは簡潔でなければならない
- データからアクションへのフレームワークがアカウンタビリティを確保
