# /training:start-2-5 - 競合分析

## 言語と品質基準

**重要**: ユーザーが使用している言語で応答してください。日本語なら日本語で、スペイン語ならスペイン語で応答してください。

---

## Claudeへの指示

包括的な競合リサーチとポジショニング戦略を教えてください。

### レッスン概要

---

**モジュール2.5: 競合分析**

競合環境を理解することはポジショニングとメッセージングに不可欠です。AgentKitsの競合を体系的にリサーチしましょう。

**所要時間:** 約30分

---

### ステップ1: 詳細な競合分析

競合コマンドを使用してください：

```
/competitor:deep "RescueTime - 個人の時間追跡と生産性ソフトウェア"
```

分析をレビューしてください：
- 会社概要
- ターゲット市場
- 製品機能
- 価格モデル
- 強み/弱み
- 市場機会

### ステップ2: 複数の競合

追加の競合を分析してください：

```
/competitor:deep "Freedom - フォーカスのためのウェブサイトとアプリブロッカー"
```

```
/competitor:deep "Focus@Will - 生産性とフォーカスのための音楽"
```

### ステップ3: SEO競合分析

SEO競合コマンドを使用してください：

```
/seo:competitor "rescuetime.com"
```

```
/seo:competitor "freedom.to"
```

レビューしてください：
- ランキングしているキーワード
- コンテンツ戦略
- バックリンクプロファイル
- テクニカルSEO

### ステップ4: バトルカードを作成する

セールスバトルカードコマンドを使用してください：

```
/sales:battlecard "RescueTime"
```

```
/sales:battlecard "Freedom"
```

バトルカードの内容をレビューしてください：
- クイック比較
- 勝利シナリオ
- 異議処理
- 証拠ポイント

### ステップ5: ポジショニングマトリックス

競合ポジショニングを作成してください：

```
AgentKitsの競合ポジショニングマトリックスを作成：

軸：
- X: 個人vsチーム重視
- Y: パッシブトラッキングvsアクティブ調整

プロット：
- RescueTime: 個人、パッシブトラッキング
- Freedom: 個人、アクティブブロッキング
- AgentKits: チーム、アクティブ調整

ポジショニングステートメント：
「フォーカスタイムを調整する必要のあるリモートチームのために、AgentKitsは同期されたディープワークを可能にするチーム生産性プラットフォームです。個人の時間だけを追跡する個人ツールとは異なります。」
```

### ステップ6: 市場調査

リサーチコマンドを使用してください：

```
/research:market "チーム生産性ソフトウェア - リモートワークツール 2024-2025"
```

```
/research:trend "リモートワーク調整 - 同期vs非同期 - チーム生産性"
```

### ステップ7: 機能比較

機能比較を作成してください：

```
機能比較表を作成：

| 機能 | AgentKits | RescueTime | Freedom |
|---------|----------|------------|---------|
| チーム調整 | ✓ コア | ✗ | ✗ |
| フォーカススケジューリング | ✓ | ✗ | ✓ |
| 時間追跡 | ✓ | ✓ コア | ✗ |
| アプリブロッキング | ✓ | ✗ | ✓ コア |
| チーム分析 | ✓ | 限定 | ✗ |
| 連携 | Slack、Zoom、GCal | 限定 | 限定 |
| 価格 | $12/ユーザー/月 | $12/月 | $8.99/月 |

ハイライト：AgentKitsが勝つ場所
```

### ステップ8: 競合対応プレイブック

対応シナリオを作成してください：

```
競合対応プレイブックを作成：

シナリオ1：RescueTimeがチーム機能をローンチ
- 即時：認識し、深さで差別化
- メッセージ：「私たちは初日からチーム向けに構築しました」
- 証拠：機能比較、顧客の声

シナリオ2：競合が価格を下げる
- 対応：価値ベースのポジショニング
- メッセージ：「チーム調整のROI vs トラッキング」
- アクション：ROIデータ付きケーススタディ

シナリオ3：新しい競合が市場参入
- 評価：機能オーバーラップ分析
- 対応：差別化要因を強化
- アクション：バトルカードを更新
```

### ステップ9: セールス資格

セールス資格を使用してください：

```
/sales:qualify "RescueTime vs AgentKitsを検討しているエンタープライズ見込み客"
```

理解してください：
- 決定基準
- ステークホルダー
- タイムライン
- 予算シグナル

### ステップ10: 差別化サマリー

ワンページサマリーを作成してください：

```
セールスチーム向けの差別化サマリーを作成：

# AgentKitsが勝つ理由

## ユニークなポジション
[ワンセンテンスポジショニング]

## トップ3差別化要因
1. チームファースト調整（証拠ポイント）
2. カレンダーツールとのリアルタイム同期（証拠ポイント）
3. チーム分析ダッシュボード（証拠ポイント）

## よくある異議 + 対応
1. 「RescueTimeの方が安い」 → [対応]
2. 「すでにFreedomを使っている」 → [対応]
3. 「別のツールは必要ない」 → [対応]

## 競合クイックリファレンス
- RescueTime: 個人トラッキング、チーム調整なし
- Freedom: ブロッキングのみ、チーム機能なし
```
