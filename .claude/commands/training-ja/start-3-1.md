# モジュール3、レッスン1：CROの基礎

## コンバージョン率最適化へようこそ

このモジュールでは、AgentKits Marketingに追加された新しいCRO（コンバージョン率最適化）スキルをマスターします。これらのスキルは、すべてのマーケティングアセットでコンバージョン率を体系的に改善するのに役立ちます。

## 学習目標

このレッスンの終わりまでに：
1. 6つのCROスキルカテゴリを理解する
2. 各CROコマンドをいつ使用するか知る
3. コンバージョンに心理学的原則を適用する
4. 最初のCRO監査を作成する

---

## CROスキルスイート

AgentKits Marketingには7つの専門CROスキルが含まれています：

| スキル | 用途 | コマンド |
|-------|---------|---------|
| `page-cro` | ランディングページ、ホームページ、価格 | `/cro:page` |
| `form-cro` | リードキャプチャ、コンタクト、デモフォーム | `/cro:form` |
| `popup-cro` | モーダル、オーバーレイ、離脱インテント | `/cro:popup` |
| `signup-flow-cro` | 登録、トライアルサインアップ | `/cro:signup` |
| `onboarding-cro` | サインアップ後のアクティベーション | `/cro:onboarding` |
| `paywall-upgrade-cro` | アプリ内ペイウォール、アップグレード | `/cro:paywall` |
| `ab-test-setup` | 実験デザイン | `/test:ab-setup` |

---

## CROフレームワーク

すべてのCRO分析はこの階層に従います：

### 1. バリュープロポジション（最高のインパクト）
- 訪問者は5秒で何を提供しているか理解できますか？
- メリットが明確ですか、機能だけでなく？

### 2. ヘッドラインの効果
- コアバリューを伝えていますか？
- 具体的で信頼できますか？

### 3. CTAの最適化
- 1つの明確な主要アクション？
- ファーストビュー、目立つ、魅力的？

### 4. トラストシグナル
- 決定の近くにソーシャルプルーフ？
- セキュリティバッジが見える？

### 5. フリクション削減
- 最小限のフォームフィールド？
- 次のステップが明確？

## 演習1：AgentKitsのランディングページを監査する

CRO原則をAgentKitsのランディングページに適用しましょう。

### 現在の状態（仮定）

**ヘッドライン：** "Team Productivity Software"
**CTA：** "Learn More"
**フォーム：** 7フィールド

### タスク

`/cro:page`コマンドを使用してCRO監査を作成：

```bash
/cro:page "AgentKitsのランディングページを分析：ヘッドライン'Team Productivity Software'、CTA'Learn More'、7フィールドフォーム。ターゲット：リモートチームマネージャー。"
```

### 期待される出力

監査は以下を特定すべき：
- 一般的なヘッドライン（具体的でもベネフィット重視でもない）
- 弱いCTA（「Learn More」vsアクション志向）
- 高いフリクション（7フィールドは多すぎる）

## 演習2：心理学を適用する

`marketing-psychology`スキルには70以上のメンタルモデルが含まれています。試してください：

```bash
/marketing:psychology "AgentKitsのトライアルサインアップ率を改善するために損失回避とソーシャルプルーフをどのように使用できますか？"
```

### CRO向けの主要モデル

| モデル | 適用 |
|-------|-------------|
| 損失回避 | 「お見逃しなく」フレーミング |
| ソーシャルプルーフ | 「10,000以上のチームが参加」 |
| アンカリング | 高価なプランを最初に表示 |
| 希少性 | 限定枠または時間 |
| 相互性 | 要求の前に無料の価値 |

## 練習課題

完全なCRO改善計画を作成：

1. **ページ監査を実行：**
   ```bash
   /cro:page "AgentKitsホームページ監査"
   ```

2. **フォームを最適化：**
   ```bash
   /cro:form "AgentKitsの7フィールドトライアルサインアップフォームを削減"
   ```

3. **A/Bテストをデザイン：**
   ```bash
   /test:ab-setup "AgentKitsの新しいヘッドラインvs現在をテスト"
   ```

作業を保存：
```
training/exercises/markit/cro/
├── landing-page-audit.md
├── form-optimization.md
└── ab-test-plan.md
```
