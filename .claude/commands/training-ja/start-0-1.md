# /training:start-0-1 - インストールとセットアップ

## 言語と品質基準

**重要**: ユーザーが使用している言語で応答してください。日本語なら日本語で、スペイン語ならスペイン語で応答してください。

---

## Claudeへの指示

学生がClaude Codeのインストールとマーケティングキットのセットアップを確認するようにガイドしてください。

### レッスン概要

次のように言ってください：

---

**モジュール0.1: インストールとセットアップ**

マーケティングワークフローに入る前に、すべてが正しくセットアップされていることを確認しましょう。

---

### ステップ1: Claude Codeを確認する

確認してもらってください：
- Claude Code内で実行している（ウェブチャットではない）
- Claude ProまたはMaxサブスクリプションを持っている

わからない場合は説明してください：
- Claude CodeはターミナルCLIバージョンです
- ファイルを直接読み書き編集できます
- claude.aiウェブチャットとは異なります

### ステップ2: マーケティングキットファイルを確認する

学生と一緒にこれらのチェックを実行してください（実際に実行）：

```
このディレクトリの内容を表示してください
```

表示されるべきもの：
- エージェント、コマンド、スキル、ワークフローを含む`.claude/`フォルダ
- ドキュメントを含む`docs/`フォルダ
- `CLAUDE.md`ファイル（プロジェクトメモリ）
- `README.md`ファイル

### ステップ3: システム構造を探索する

マーケティングキットの構造を見せてください：

```
.claude/内のすべてのフォルダをリストする
```

各コンポーネントを説明してください：
- `agents/` - 18の専門マーケティングエージェント
- `commands/` - 機能別に整理された76のスラッシュコマンド
- `skills/` - マーケティングドメイン知識
- `workflows/` - コアマーケティング、セールス、CRMワークフロー

### ステップ4: 利用可能なコマンドを探索する

コマンドカテゴリを見せてください：

```
.claude/commands/内のすべてのフォルダをリストする
```

主要なコマンドグループを説明してください：
- `campaign/` - `/campaign:plan`、`/campaign:brief`、`/campaign:analyze`
- `content/` - `/content:blog`、`/content:social`、`/content:email`、`/content:landing`
- `seo/` - `/seo:keywords`、`/seo:audit`、`/seo:optimize`
- `analytics/` - `/analytics:roi`、`/analytics:funnel`、`/analytics:report`
- `sales/` - `/sales:pitch`、`/sales:outreach`、`/sales:battlecard`

### ステップ5: 最初のコマンドをテストする

実際のコマンドを試してもらってください：

```
/brainstorm "B2B SaaS製品に最適なマーケティングチャネルは何ですか？"
```

最初のコマンド実行を祝いましょう！

### ステップ6: ドキュメントを確認する

主要なドキュメントを見せてください：

```
docs/usage-guide.mdを読む（最初の50行）
```

説明してください：
- `docs/usage-guide.md` - 完全なシステムリファレンス
- `docs/brand-guidelines.md` - ブランド基準テンプレート
- `docs/content-style-guide.md` - ライティング基準
- `docs/campaign-playbooks.md` - キャンペーンテンプレート
- `docs/channel-strategies.md` - プラットフォーム戦術
- `docs/analytics-setup.md` - トラッキング設定

### 次は

伝えてください：
