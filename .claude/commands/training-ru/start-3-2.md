# Модуль 3, Урок 2: Оптимизация форм и регистрации

## Мастерство CRO для форм и регистрации

Формы — это привратники конверсии. Каждое лишнее поле стоит вам лидов. Этот урок научит вас оптимизировать формы захвата лидов и процессы регистрации.

## Цели обучения

По завершении этого урока вы сможете:
1. Применять правило максимум 5 полей
2. Оптимизировать процессы регистрации для конверсии
3. Систематически снижать трение в формах
4. Проектировать многошаговые прогрессивные формы

---

## Принципы CRO для форм

### Правило 5 полей

Каждое поле сверх 5 снижает конверсию примерно на ~10%.

**Только необходимые поля:**
1. Email (всегда обязательно)
2. Имя (иногда)
3. Компания (только B2B)
4. Пароль (только при регистрации)
5. Один квалификатор при необходимости

**Отложите всё остальное** на период после регистрации.

### Точки трения, которые нужно устранить

| Трение | Решение |
|--------|---------|
| Слишком много полей | Удалить или отложить |
| Требования к паролю | Показывать встроенно, а не после ошибки |
| Обязательный телефон | Сделать необязательным или удалить |
| CAPTCHA | Использовать невидимые альтернативы |
| Нет социальной авторизации | Добавить Google/SSO опции |

---

## Команда CRO для форм

Используйте `/cro:form` для форм захвата лидов:

```bash
/cro:form "Optimize AgentKits's demo request form: Name, Email, Company, Phone, Title, Team Size, Message"
```

### Ожидаемые рекомендации

1. **Удалить:** Message (спросить при последующем контакте)
2. **Удалить:** Phone (можно получить позже)
3. **Сделать необязательным:** Title
4. **Оставить:** Name, Email, Company, Team Size

Сокращение с 7 → 4 полей = ожидаемый рост конверсии на +30%

---

## Оптимизация процесса регистрации

Для регистрации аккаунта используйте `/cro:signup`:

```bash
/cro:signup "Analyze AgentKits's trial signup: Email → Password → Company → Team Size → Use Case → Payment"
```

### Паттерны процесса регистрации

| Паттерн | Подходит для | Конверсия |
|---------|--------------|-----------|
| Начало только с email | Высшая конверсия | Начать с email, прогрессивное профилирование |
| Сначала социальные сети | Потребительские приложения | Google/SSO на видном месте |
| Одностраничный минимальный | B2C, простые продукты | Все поля видны |
| Многошаговый с прогрессом | B2B, сложные | Управляемый, показывает прогресс |

## Упражнение 1: Аудит формы

Проведите аудит текущей формы запроса демо AgentKits:

```bash
/cro:form "7-field form: Name, Email, Company, Phone, Job Title, Team Size, Message. Goal: schedule demo calls."
```

Создайте рекомендации в:
```
training/exercises/markit/cro/form-audit.md
```

### Ваш результат должен включать

1. Поля для удаления
2. Поля для преобразования в необязательные
3. Стратегия прогрессивного профилирования
4. Улучшения текста (текст кнопки, метки)

---

## Упражнение 2: Редизайн процесса регистрации

Спроектируйте оптимизированный процесс регистрации для AgentKits:

```bash
/cro:signup "Design optimal trial signup for AgentKits. Current: 5-step process. Goal: maximize trial starts."
```

Учтите:
- Первоначальный захват только email
- Опции OAuth (Google Workspace для B2B)
- Когда собирать информацию о компании
- Разделение онбординга и регистрации

---

## Упражнение 3: Проектирование многошаговой формы

Для сложных B2B регистраций спроектируйте многошаговый подход:

**Шаг 1:** Только email
**Шаг 2:** Компания + размер команды (с индикатором прогресса)
**Шаг 3:** Необязательный выбор варианта использования

Используйте навык form-cro для валидации:

```bash
/cro:form "Design 3-step progressive form for AgentKits enterprise demo"
```
