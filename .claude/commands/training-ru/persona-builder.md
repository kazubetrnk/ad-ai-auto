# /training-ru:persona-builder - Интерактивный конструктор персон

## Языковые стандарты и стандарты качества

**КРИТИЧНО**: Отвечайте на том же языке, который использует пользователь. Если вьетнамский, отвечайте на вьетнамском. Если испанский, отвечайте на испанском.

---

## Инструкции для Claude

Проведите пользователей без опыта разработки через создание персоны покупателя шаг за шагом, используя **интерактивный UX-шаблон**. Задавайте вопросы с 2-4 кликабельными вариантами на каждом шаге. Это практическое учебное занятие для начинающих.

### Приветственное сообщение

---

**Интерактивный конструктор персон**

Я помогу вам создать детальную персону покупателя для вашего продукта или услуги. Опыт в маркетинге не требуется - просто отвечайте на несколько вопросов, выбирая из предложенных мной вариантов.

**Что вы создадите:**
- Полный профиль персоны покупателя
- Ключевые месседжи для этой персоны
- Рекомендации по каналам коммуникации

**Длительность:** ~15 минут

Начнем!

---

### Шаг 1: Тип бизнеса

**ВАЖНО**: Используйте инструмент AskUserQuestion, чтобы спросить:

**Вопрос:** "Какой тип продукта или услуги вы продвигаете?"

**Варианты:**
1. **SaaS / Программное обеспечение** - Облачное ПО, приложения, цифровые инструменты
2. **Электронная коммерция** - Физические товары, интернет-магазин
3. **Профессиональные услуги** - Консалтинг, агентство, коучинг
4. **Другое** - Позвольте пользователю указать

---

### Шаг 2: Целевая аудитория

**ВАЖНО**: Используйте инструмент AskUserQuestion, чтобы спросить:

**Вопрос:** "Кто ваша основная целевая аудитория?"

**Варианты:**
1. **B2B лица, принимающие решения** - Менеджеры, директора, руководители в компаниях (Рекомендуется для SaaS)
2. **B2B конечные пользователи** - Отдельные сотрудники, члены команды
3. **B2C потребители** - Индивидуальные потребители для личного использования
4. **Другое** - Позвольте пользователю указать

---

### Шаг 3: Размер компании (если B2B)

Если был выбран B2B, используйте инструмент AskUserQuestion:

**Вопрос:** "На какой размер компании вы обычно ориентируетесь?"

**Варианты:**
1. **Стартапы** - 1-20 сотрудников, основатели/первая команда
2. **Малый и средний бизнес** - 20-200 сотрудников, растущие команды (Рекомендуется)
3. **Средний рынок** - 200-2000 сотрудников, руководители отделов
4. **Корпорации** - 2000+ сотрудников, вовлечен отдел закупок

---

### Шаг 4: Основная проблема

**ВАЖНО**: Используйте инструмент AskUserQuestion:

**Вопрос:** "Какая проблема №1 решается вашим продуктом?"

**Варианты:**
1. **Экономия времени** - Автоматизация, эффективность, продуктивность
2. **Экономия денег** - Снижение затрат, лучший ROI
3. **Снижение рисков** - Соответствие требованиям, безопасность, надежность
4. **Рост дохода** - Больше продаж, лидов, клиентов

---

### Шаг 5: Критерии принятия решения

**ВАЖНО**: Используйте инструмент AskUserQuestion:

**Вопрос:** "Что наиболее важно для ваших покупателей при выборе решения?"

**Варианты:**
1. **Цена / Ценность** - Внимание к бюджету, фокус на ROI
2. **Функции / Возможности** - Опытные пользователи, специфические потребности (Рекомендуется)
3. **Простота использования** - Нетехнические пользователи, быстрое освоение
4. **Доверие / Бренд** - Устоявшиеся игроки, рекомендации

---

### Шаг 6: Шаблон персоны

**ВАЖНО**: Используйте инструмент AskUserQuestion:

**Вопрос:** "Какой архетип персоны лучше всего подходит вашему идеальному клиенту?"

**Варианты:**
1. **Менеджер Мария** - B2B менеджер, руководитель команды, ориентирован на результат (Рекомендуется для B2B)
2. **Стартапер Сэм** - Основатель, выполняет много ролей, ориентирован на рост
3. **Соло Стив** - Индивидуальный предприниматель, внимателен к бюджету, делает сам
4. **Настраиваемая** - Создать с нуля на основе предыдущих ответов

---

### Генерация персоны

На основе всех ответов сгенерируйте полную персону, используя этот формат:

```markdown
## [Имя персоны]
**Роль:** [Должность на основе ответов]
**Компания:** [Тип/Размер на основе ответов]

### Демографические данные
- Возраст: [Подходящий диапазон]
- Образование: [Подходящий уровень]
- Опыт: [Лет на должности]
- Подчиняется: [Кому подчиняется]

### Цели
1. [Основная цель, соответствующая проблеме]
2. [Вторичная цель, соответствующая критериям решения]
3. [Карьерная/личная цель]

### Задачи
1. [Основная проблема из Шага 4]
2. [Связанная задача]
3. [Препятствие для достижения целей]

### Как помогает [Продукт]
- Решает [проблема 1] с помощью [конкретное решение]
- Обеспечивает [цель] через [функция]
- Снижает [задача] с помощью [преимущество]

### Возражения и ответы
- "[Беспокойство о бюджете]" → [Ответ, ориентированный на ценность]
- "[Время на внедрение]" → [Ответ о простоте освоения]
- "[Риск изменений]" → [Ответ, укрепляющий доверие]

### Предпочтительные каналы
- **Поиск информации:** [Где они исследуют]
- **Контент:** [Что они потребляют]
- **Социальные сети:** [Где они общаются]

### Резонирующие месседжи
- Начать с: [Основное преимущество]
- Подчеркнуть: [Ключевое отличие]
- Доказать с помощью: [Тип доказательства]

### Характерная цитата
"[Утверждение, которое отражает их образ мышления]"
```

---

### Подтверждение и сохранение

После генерации спросите:

**Вопрос:** "Хотите ли вы сохранить эту персону?"

**Варианты:**
1. **Сохранить в docs/** - Сохранить как `docs/personas/[имя].md`
2. **Доработать дальше** - Скорректировать конкретные разделы
3. **Создать другую персону** - Начать заново для другого сегмента
4. **Готово** - Завершить обучение

## Ключевые моменты обучения

1. **Интерактивный UX-шаблон**: Всегда используйте AskUserQuestion с 2-4 вариантами
2. **Шаблоны помогают новичкам**: Предлагайте рекомендуемые варианты с меткой (Рекомендуется)
3. **Постепенное построение**: Каждый ответ определяет следующий вопрос
4. **Подтверждение перед действием**: Спрашивайте перед сохранением или важными действиями
5. **Празднуйте завершение**: Отметьте их достижение
6. **Предоставляйте следующие шаги**: Направляйте их к связанным командам
