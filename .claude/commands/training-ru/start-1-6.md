### Обзор урока

---

**Модуль 1.6: Память проекта**

CLAUDE.md — это как предоставление Claude постоянного инструктивного документа. Каждый раз, когда вы работаете над этим проектом, Claude сначала читает этот файл и применяет указанные в нём рекомендации.

**Продолжительность:** ~20 минут

---

### Шаг 1: Покажите текущий CLAUDE.md

Прочитайте файл CLAUDE.md проекта:

```
Read the CLAUDE.md file in this project
```

Пройдитесь по каждому разделу:
- Роль и обязанности
- Рабочие процессы (Маркетинг, Продажи, CRM)
- Маркетинговые агенты
- Каталог навыков
- Категории команд
- Управление документацией

### Шаг 2: Объясните, как это работает

Когда существует CLAUDE.md, Claude автоматически:
- Знает, какие агенты доступны
- Понимает структуру рабочих процессов
- Ссылается на соответствующие команды
- Следует правилам маркетинга
- Использует правильные навыки

Вам не нужно напоминать Claude каждый раз — это автоматически!

### Шаг 3: Ключевые разделы CLAUDE.md

Объясните критически важные разделы:

**Рабочие процессы:**
```markdown
### Core Workflows
- **Marketing:** `./.claude/workflows/primary-workflow.md`
- **Sales:** `./.claude/workflows/sales-workflow.md`
- **CRM:** `./.claude/workflows/crm-workflow.md`
```

**Сопоставление агентов:**
```markdown
### Core Marketing Agents
- `attraction-specialist` - TOFU (SEO, landing pages)
- `lead-qualifier` - Intent detection, scoring
- `email-wizard` - Sequences, automation
...
```

**Категории команд:**
```markdown
### Campaign Management
- `/campaign:plan`, `/campaign:brief`, `/campaign:analyze`

### Content Creation
- `/content:blog`, `/content:social`, `/content:email`
...
```
