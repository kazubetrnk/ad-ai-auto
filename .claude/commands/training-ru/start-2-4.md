## Инструкции для Claude

Обучайте анализу данных, извлечению инсайтов и составлению отчетов для руководства с использованием аналитических команд.

### Обзор урока

---

**Модуль 2.4: Анализ данных кампании**

Анализ данных часто занимает много времени. Давайте освоим превращение данных в практические инсайты и убедительные отчеты.

**Продолжительность:** ~35 минут

---

### Шаг 1: Анализ ROI

Используйте аналитические команды:

```
/analytics:roi "Q1 campaign - $50K spend across LinkedIn, Google, Email"
```

Проверьте расчет ROI:
- Общие затраты по каналам
- Приписанная выручка
- ROAS по каналам
- Стоимость привлечения клиента

### Шаг 2: Анализ воронки

Анализируйте воронку конверсии:

```
/analytics:funnel "trial signup - visitor to trial to paid conversion"
```

Проверьте метрики воронки:
- Трафик по источникам
- Коэффициенты конверсии на каждом этапе
- Точки оттока
- Возможности для оптимизации

### Шаг 3: Отчеты о результатах

Генерируйте отчеты о результатах:

**Еженедельный отчет:**
```
/report:weekly "AgentKits" "current week"
```

**Ежемесячный отчет:**
```
/report:monthly "AgentKits" "current month"
```

### Шаг 4: Производительность каналов

Анализируйте по каналам:

```
/analytics:report "channel performance" "LinkedIn, Google, Email, Organic"
```

Создайте сравнение каналов:
- Вклад в трафик
- Качество лидов
- Коэффициенты конверсии
- Эффективность затрат

### Шаг 5: Производительность контента

Анализируйте эффективность контента:

```
/analytics:report "content performance" "blog posts, landing pages, email sequences"
```

Ключевые метрики:
- Трафик по единицам контента
- Вовлеченность (время, прокрутка, репосты)
- Коэффициент конверсии
- Качество лидов

### Шаг 6: Анализ качества лидов

Используйте оценку лидов для анализа:

```
/crm:score "analyze lead quality by source and campaign"
```

Проверьте:
- Процент MQL по источникам
- Конверсия SQL по кампаниям
- Тренды средней оценки лидов

### Шаг 7: Резюме для руководства

Создайте резюме для руководства:

```
Create an executive summary of Q1 marketing performance:

STRUCTURE:
1. Headline metrics (vs targets)
2. Top 3 wins with data
3. Top 3 challenges with impact
4. Channel performance snapshot (table)
5. Key learnings (3 insights)
6. Q2 recommendations (prioritized)
7. Budget request with justification

Keep it to ONE PAGE maximum.
```

### Шаг 8: Фреймворк от данных к действиям

Обучите фреймворку инсайтов:

```
For each finding, document:

1. OBSERVATION: What does the data show?
2. INSIGHT: Why is this happening?
3. IMPLICATION: What does it mean?
4. RECOMMENDATION: What should we do?
5. EXPECTED IMPACT: What will change?
```

### Шаг 9: Операционные чеклисты

Используйте аналитические чеклисты:

```
/checklist:analytics-monthly "current month" "AgentKits"
```

Проверьте ежемесячные аналитические задачи:
- Проверка качества данных
- Верификация платформы
- Точность отчетности
- Валидация атрибуции

### Шаг 10: Шаблоны отчетов

Объясните повторно используемую отчетность:

```
Weekly Report Workflow:
1. /analytics:roi "campaign" - Calculate ROI
2. /analytics:funnel "funnel" - Analyze funnel
3. /report:weekly "client" "week" - Generate report

Monthly Report Workflow:
1. /analytics:report "all channels" - Full analysis
2. /crm:score "lead quality" - Lead analysis
3. /report:monthly "client" "month" - Generate report
```

## Ключевые моменты обучения
- Команды `/analytics:*` анализируют производительность
- Команды `/report:*` генерируют отчеты
- Анализ ROI и воронки являются фундаментальными
- Резюме для руководства должны быть краткими
- Фреймворк от данных к действиям обеспечивает подотчетность
