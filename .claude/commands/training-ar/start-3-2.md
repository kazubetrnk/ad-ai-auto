# الوحدة 3، الدرس 2: تحسين النماذج والتسجيل

## إتقان تحسين معدل التحويل للنماذج والتسجيل

النماذج هي بوابات التحويل. كل حقل غير ضروري يكلفك عملاء محتملين. يعلمك هذا الدرس كيفية تحسين نماذج التقاط العملاء المحتملين وتدفقات التسجيل.

## أهداف التعلم

بنهاية هذا الدرس، ستتمكن من:
1. تطبيق قاعدة الحد الأقصى 5 حقول
2. تحسين تدفقات التسجيل لزيادة التحويل
3. تقليل احتكاك النماذج بشكل منهجي
4. تصميم نماذج تدريجية متعددة الخطوات

---

## مبادئ تحسين معدل التحويل للنماذج

### قاعدة الـ 5 حقول

كل حقل يتجاوز 5 حقول يقلل التحويل بنسبة ~10%.

**الحقول الأساسية فقط:**
1. البريد الإلكتروني (مطلوب دائماً)
2. الاسم (أحياناً)
3. الشركة (B2B فقط)
4. كلمة المرور (التسجيل فقط)
5. مؤهل واحد إذا لزم الأمر

**أجل كل شيء آخر** إلى ما بعد التسجيل.

### نقاط الاحتكاك التي يجب إزالتها

| الاحتكاك | الحل |
|----------|-----|
| عدد كبير جداً من الحقول | احذف أو أجل |
| متطلبات كلمة المرور | اعرضها مباشرة، وليس بعد الخطأ |
| رقم الهاتف مطلوب | اجعله اختيارياً أو احذفه |
| CAPTCHA | استخدم بدائل غير مرئية |
| عدم وجود تسجيل دخول اجتماعي | أضف خيارات Google/SSO |

---

## أمر تحسين معدل التحويل للنماذج

استخدم `/cro:form` لنماذج التقاط العملاء المحتملين:

```bash
/cro:form "Optimize AgentKits's demo request form: Name, Email, Company, Phone, Title, Team Size, Message"
```

### التوصيات المتوقعة

1. **احذف:** الرسالة (اسأل في المتابعة)
2. **احذف:** الهاتف (يمكن التقاطه لاحقاً)
3. **اجعله اختيارياً:** المسمى الوظيفي
4. **احتفظ بـ:** الاسم، البريد الإلكتروني، الشركة، حجم الفريق

تم التقليل من 7 → 4 حقول = زيادة مقدرة بـ +30% في التحويلات

---

## تحسين تدفق التسجيل

لتسجيل الحسابات، استخدم `/cro:signup`:

```bash
/cro:signup "Analyze AgentKits's trial signup: Email → Password → Company → Team Size → Use Case → Payment"
```

### أنماط تدفق التسجيل

| النمط | الأفضل لـ | التحويل |
|---------|----------|------------|
| البدء بالبريد الإلكتروني فقط | أعلى معدل تحويل | ابدأ بالبريد الإلكتروني، ثم التوصيف التدريجي |
| الأولوية للتسجيل الاجتماعي | تطبيقات المستهلكين | Google/SSO بارز |
| صفحة واحدة بحد أدنى | B2C، منتجات بسيطة | جميع الحقول مرئية |
| متعدد الخطوات مع مؤشر التقدم | B2B، معقد | موجه، يعرض التقدم |

## التمرين 1: تدقيق النموذج

قم بتدقيق نموذج طلب العرض التوضيحي الحالي لـ AgentKits:

```bash
/cro:form "7-field form: Name, Email, Company, Phone, Job Title, Team Size, Message. Goal: schedule demo calls."
```

أنشئ التوصيات في:
```
training/exercises/markit/cro/form-audit.md
```

### يجب أن يتضمن ناتجك

1. الحقول المراد حذفها
2. الحقول المراد جعلها اختيارية
3. استراتيجية التوصيف التدريجي
4. تحسينات النسخ (نص الزر، التسميات)

---

## التمرين 2: إعادة تصميم تدفق التسجيل

صمم تدفق تسجيل محسّن لـ AgentKits:

```bash
/cro:signup "Design optimal trial signup for AgentKits. Current: 5-step process. Goal: maximize trial starts."
```

ضع في الاعتبار:
- التقاط البريد الإلكتروني الأولي فقط
- خيارات OAuth (Google Workspace لـ B2B)
- متى يتم جمع معلومات الشركة
- فصل الإعداد عن التسجيل

---

## التمرين 3: تصميم نموذج متعدد الخطوات

للتسجيلات المعقدة في B2B، صمم نهجاً متعدد الخطوات:

**الخطوة 1:** البريد الإلكتروني فقط
**الخطوة 2:** الشركة + حجم الفريق (مع شريط التقدم)
**الخطوة 3:** اختيار حالة الاستخدام الاختيارية

استخدم مهارة form-cro للتحقق:

```bash
/cro:form "Design 3-step progressive form for AgentKits enterprise demo"
```
