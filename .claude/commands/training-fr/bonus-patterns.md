## Qu'est-ce qu'une Bibliothèque de Modèles ?

Une bibliothèque de modèles documente :
- **Modèles gagnants** - Ce qui a fonctionné et pourquoi
- **Modèles défaillants** - Ce qui n'a pas fonctionné et pourquoi
- **Modèles spécifiques aux personas** - Ce qui résonne avec chaque audience
- **Modèles spécifiques aux canaux** - Ce qui fonctionne sur chaque plateforme
- **Modèles saisonniers** - Ce qui fonctionne à différents moments

## Headline Patterns

### High-Performers (2x+ average)
| Pattern | Example | Performance | Context |
|---------|---------|-------------|---------|
| How [Persona] can [Benefit] | "How remote managers save 10 hrs/week" | 2.3x CTR | Works best for Solo Sam |

| Pattern | Why It Failed |

## Utiliser les Commandes pour Construire des Modèles

Après chaque campagne, utilisez :

```
/analytics:report "campaign name" "all metrics"
```

Ensuite documentez :
1. Quel contenu a le mieux performé ?
2. Quels titres ont obtenu le CTR le plus élevé ?
3. Quels CTA ont généré le plus de conversions ?
4. Quels horaires/jours ont le mieux fonctionné ?

## Meilleures Pratiques pour la Bibliothèque de Modèles

1. **Examiner avant les campagnes** - Vérifiez ce qui a fonctionné auparavant
2. **Mettre à jour après les campagnes** - Ajoutez de nouveaux apprentissages
3. **Inclure les échecs** - Sachez ce qu'il faut éviter
4. **Ajouter du contexte** - Comprendre pourquoi ça a fonctionné est important
5. **Suivre les métriques** - Quantifiez la performance

## Modèles CRO

### Modèles de Page de Destination

| Élément | Modèle Gagnant | Amélioration |
|---------|-----------------|-------------|
| Titre | Bénéfice spécifique + nombre | +40% engagement |
| Sous-titre | "Comment" ou "Pour qui" | +25% clarté |
| CTA | Première personne ("Commencer Mon Essai") | +28% clics |
| Champs de formulaire | 3-4 champs max | +35% complétions |
| Preuve sociale | Logos + "Rejoignez X équipes" | +22% confiance |

### Modèles de Formulaire

| Modèle | Impact |
|---------|--------|
| Disposition en une seule colonne | +15% complétion |
| Indicateur de progression (multi-étapes) | +20% complétion |
| Validation en ligne | +22% réduction des erreurs |
| Valeurs par défaut intelligentes | +10% complétion |
| "Aucune carte de crédit requise" | +18% inscriptions |

### Modèles de Popup

| Type | Timing | Conversion |
|------|--------|------------|
| Intention de sortie | Souris quitte la zone d'affichage | 3-5% capture |
| Défilement 50% | Visiteurs engagés | 2-4% capture |
| Délai de 30s | Après lecture | 1-3% capture |

**Éléments de Popup à Haute Conversion :**
- Valeur claire dans le titre (pas "S'abonner")
- Champ unique (email seulement)
- Bouton de fermeture visible
- Compatible mobile (plein écran OK)
- Limite de fréquence (une fois par session)

### Modèles de Flux d'Inscription

| Modèle | Impact sur la Conversion |
|---------|-------------------|
| Début avec email uniquement | +45% capture initiale |
| Option Google/SSO | +30% pour le grand public |
| Profilage progressif | +25% vs formulaire long |
| Option afficher le mot de passe | +12% complétions |

### Modèles d'Onboarding

| Élément | Impact sur l'Activation |
|---------|-------------------|
| État vide avec valeur | +35% taux d'action |
| Données/modèles d'exemple | +28% exploration |
| Liste de contrôle en 3 étapes | +40% complétion |
| Moment aha en <5 min | +50% conversion d'essai |

### Modèles de Paywall

| Déclencheur | Conversion |
|---------|------------|
| Après moment de valeur | Plus élevé (2-4%) |
| Restriction de fonctionnalité | Moyen (1-2%) |
| Limite d'utilisation | Variable |
| Expiration de l'essai | Dépend de l'engagement |

**Éléments Efficaces d'Écran de Mise à Niveau :**
- Tableau de comparaison des fonctionnalités
- Plan actuel mis en évidence
- Témoignage d'un utilisateur similaire
- Badge "Le plus populaire"
- Remise annuelle mise en avant
